场景1

最近在写ASP.NET Core MVC项目，发现在处理异常的时候会遇到duplicate code，例如这样:

var client = new HttpClient { Timeout = TimeSpan.FromSeconds(3) };
try
{
    // code here...
    var response = await client.PostAsync(ApiUrls.LoginUrl, content);
    if (response.IsSuccessStatusCode)
    {
        // code here...
    }
}
catch(HttpRequestException ex)
{
    ViewData["Message"] = "❌连接错误";
    return View(); // duplicate
}
catch (Exception ex)
{
    ViewData["Message"] = "❌账号或密码错误";
    return View(); // deuplicate
}
finally
{
    client.Dispose();
}

可以看到，在异常内部我们根据不同的错误给出不同Message，在这个场景里我们只要关注Message的不同。利用 Pattern Matching(since C#7.0) 就可以优化return View();这一行，修改后的代码如下

var client = new HttpClient { Timeout = TimeSpan.FromSeconds(3) };
try
{
    // code here...
    if (response.IsSuccessStatusCode)
    {
        // code here...
    }
    else
    {
        throw new Exception();
    }
}
catch (Exception excetion)
{
    switch (excetion)
    {
        case HttpRequestException hrex:
            ViewData["Message"] = "❌连接错误";
            break;
        case Exception ex:
            ViewData["Message"] = "❌账号或密码错误";
            break;
    }
    return View(); // not duplicate now
}
finally
{
    client.Dispose();
}

试想一下如果return View();这行代码在其他场景中等价于很多代码，而且也不好用 extract method 进行处理的时候，Pattern Matching 不妨是一个不错的选择。

场景2

在WinForm中空间的Tag属性可以作为数据的媒介，假设在Do方法内部只针对ListView.CheckedListViewItemCollection这个类型的集合做遍历处理，常规的代码可能如下

private void Do(object sender, EventArgs e)
{
    var items = Tag as ListView.CheckedListViewItemCollection;
    if(items == null) return;
    for(var i = items.Count - 1; i >=0; i--)
    {
        // code here...
    }
}

而利用 Pattern Matching 可以这样写

private void Do(object sender, EventArgs e)
{
    if(!(Tag is ListView.CheckedListViewItemCollection) items) return;
    for(var i = items.Count - 1; i >= 0; i--)
    {
        // code here...
    }
}
